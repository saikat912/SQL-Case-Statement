# SQL CASE Statements

This repository contains my solutions and practice exercises completed during the **SQL CASE Statements** guided project on Coursera. The project demonstrates how to use SQL `CASE` statements to perform conditional logic, categorize data, combine them with aggregate functions and joins, and generate meaningful business insights using the **Employees** sample database.

## 📖 Project Overview

The SQL `CASE` statement is a powerful conditional expression that allows you to return different values based on specified conditions. Throughout this project, I applied `CASE` statements in real-world SQL scenarios, including employee classification, salary analysis, and data transformation.

## 🎯 Learning Objectives

- Understand the purpose and syntax of SQL `CASE` statements.
- Write simple and complex `CASE` expressions.
- Combine `CASE` statements with aggregate functions.
- Use `CASE` statements with SQL joins.
- Perform conditional aggregation and data transposition.

## 📂 Project Tasks

### Task 1: SQL CASE Statement
- Retrieve employee records
- Convert gender values (`M` → Male, `F` → Female)

### Task 2: Multiple CASE Conditions
- Categorize customers by age
- Classify employees based on employment year

### Task 3: CASE with Aggregate Functions
- Calculate average salaries
- Categorize salary ranges
- Count employees in each salary category

### Task 4: CASE with SQL Joins
- Join employee and manager tables
- Identify managers and regular employees

### Task 5: CASE with Joins and Aggregates
- Analyze salary history
- Calculate salary differences
- Categorize salary increases
- Determine current employee status

### Task 6: Data Transposition
- Summarize sales by profit category
- Pivot department employee counts using `CASE`
